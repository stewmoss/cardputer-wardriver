# M5 Cardputer Wardriver — v0.5.1 Release Notes

**Release date:** April 18, 2026
**Target hardware:** M5Stack Cardputer v1.1 (ESP32-S3)
**Firmware binary:** `wardriver-0.5.1.bin`

---

## Highlights

- New **Channel Activity** dashboard (View E) — a real-time per-channel bar chart with three view modes and congestion colour-coding
- Dashboard count increased from five views to **six** (A → B → C → D → E → F)
- New **SPACE** key for cycling dashboard sub-modes
- Refreshed startup splash screen
- Consistent footer text across all views

---

## New Features

### Channel Activity Dashboard (View E)

A vertical bar chart showing WiFi AP counts for channels 1–13, with three modes cycled using the **SPACE** key:

| Mode | Description |
|------|-------------|
| **Session** (default) | Accumulated AP counts across all scan sweeps since boot |
| **Sweep** | AP counts from the most recently completed scan only |
| **Unique** | Counts only unique BSSIDs per channel |

**Visual design:**

- Bars are colour-coded by relative congestion: **green** (≤25%), **yellow** (≤50%), **orange** (≤75%), **red** (>75%)
- Channels with zero APs show a subtle grey outline indicator
- Y-axis auto-scales with dotted grid lines at 0%, 25%, 50%, 75%, and 100% of the peak value
- Incremental bar updates — only changed bars are redrawn (no flicker)
- Scale changes trigger a full chart redraw with updated Y-axis labels

### SPACE Key — Dashboard Sub-Mode

Pressing **SPACE** during a session now cycles the sub-mode of the current dashboard view. Currently used by Dashboard E to switch between Session, Sweep, and Unique channel views.

## Keyboard Controls

| Key | Action |
|-----|--------|
| `ENTER` | Cycle dashboard views (A → B → C → D → E → F) |
| `SPACE` | Toggle dashboard sub-mode (where applicable) |
| `H` | Show/hide keyboard help overlay |
| `B` | Toggle display blank |
| `C` | Enter Config Mode |
| `S` | Toggle sound mute |
| `Q` | Safe shutdown with session summary |
| `X` | Stop/start WiFi scanning |
| `P` | Pause/resume CSV logging |
| `G0 button` | Boot: force Config Mode · Running: drop FLAG marker · Shutdown: reboot |

## Technical Details

### Per-Channel Tracking (Config.h / ScanSession)

Three new `uint16_t[13]` arrays added to `ScanSession`:

| Array | Purpose |
|-------|---------|
| `channelCountsSweep` | Per-channel AP counts from the latest completed sweep |
| `channelCountsSession` | Accumulated per-channel counts across all sweeps |
| `channelCountsUnique` | Per-channel counts for unique BSSIDs only |

Channel counts are populated in `processCompletedScan()` (sweep and session) and during BSSID deduplication (unique).

### Display Cache (DashECache)

Dashboard E uses an incremental update cache (`DashECache`) storing per-bar pixel heights, colours, the current Y-axis maximum, and active view mode. Bars are updated with a delta approach — only the height difference is drawn or erased. A full redraw is triggered only on view mode switch or Y-axis scale change.
