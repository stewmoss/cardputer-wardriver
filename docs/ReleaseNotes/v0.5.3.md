# M5 Cardputer Wardriver — v0.5.3 Release Notes

**Target hardware:** M5Stack Cardputer v1.1 and Cardputer ADV (ESP32-S3)
**Firmware binary:** `wardriver-0.5.3.bin`

---

## Highlights

- **Manual WiGLE upload** can now be triggered on demand from the **shutdown screen** by pressing `U` — perform an upload before powering off without rebooting back into a scan session.
- New **Minimum-sweeps quarantine** pre-screens CSV files before upload and moves "thin" sessions to `/wardriver/uploaded/thin/` instead of submitting low-value data to WiGLE.
- New `min_sweeps_threshold` and `retry_thin_files` options exposed in `wardriverconfig.txt`, the Upload tab of the web portal, and `/status` JSON.
- Structured system telemetry now writes to `/wardriver/logs/stats.csv` for heap, PSRAM, battery, scan, CPU, temperature, RTOS, and loop-timing analysis.

---

## New Features

### Manual Upload from the Shutdown Screen

After pressing `Q` to safely shut down, the locked shutdown screen now shows a `U=Upload` hint in its footer when upload is configured and pending CSV files exist.

- Press `U` to start the upload immediately. The device re-mounts the SD card and re-enables WiFi STA, runs the same upload pipeline as auto-upload (sweep pre-scan → upload → summary), and **returns to the locked shutdown screen** when finished. Only one upload per shutdown is allowed.
- Press `ESC` during upload to cancel and return to the locked shutdown screen.
- Press `G0` at any time to reboot, or `B` to blank the screen.

The `U` hint is hidden if upload is not configured, no pending CSV files exist, or shutdown was triggered by low battery.

### Minimum-Sweeps Quarantine

A "sweep" is one full pass through the WiFi channel list, identified by a unique `FirstSeen` timestamp in the CSV. CSV files whose distinct-sweep count is below `min_sweeps_threshold` (default `20`) are skipped during upload and moved to `/wardriver/uploaded/thin/`.

- Set `min_sweeps_threshold` to `0` to disable the check entirely.
- Enable `retry_thin_files` to re-evaluate quarantined files on the next upload run (useful after raising or lowering the threshold).
- The completion summary now shows a **Skipped** row reporting how many files were quarantined.

Each file is pre-scanned in a single blocking pass before upload begins. A **Checking sweeps…** screen is shown during the check, and the scan exits early once the threshold is met so typical files resolve quickly.

### System Stats CSV Logging

System stats logging now writes structured telemetry rows to `/wardriver/logs/stats.csv` instead of relying only on formatted debug text. This makes long-run diagnostics easier to graph and compare across sessions.

- Controlled by the existing `debug.enabled`, `debug.system_stats_enabled`, and `debug.system_stats_interval_s` settings.
- Adds lifecycle rows for `init`, periodic `run`, `pre_upload`, and `exit` events.
- Captures timestamp/epoch when available, uptime, heap and PSRAM counters, scan and station counters, battery percentage and millivolts, charging state, internal temperature, CPU frequency, RTOS task data, and maximum loop time.
- `stats.csv` is diagnostic output only and should not be uploaded to WiGLE.

## Configuration

### New Upload Fields

| Key | Type | Default | Description |
|-----|------|---------|-------------|
| `min_sweeps_threshold` | int | `20` | Minimum distinct sweeps required to upload. `0` disables the check. |
| `retry_thin_files` | bool | `false` | Re-evaluate quarantined files on next upload run. Ignored when threshold is `0`. |

Both fields are editable from the **Upload** tab of the web portal. The "Retry quarantined files" checkbox is automatically disabled in the UI when the threshold is `0`.

### Debug Output

No new config keys are required for stats CSV logging. Enable `debug.enabled` and `debug.system_stats_enabled` to create `/wardriver/logs/stats.csv`; use `debug.system_stats_interval_s` to control the periodic `run` row interval.

## Keyboard Controls

| Key | Action |
|-----|--------|
| **U** | Trigger manual WiGLE upload (shutdown screen only, when upload is configured and pending files exist) |

---

## Upgrade Notes

- Existing configs upgrade safely. `min_sweeps_threshold` defaults to `20` and `retry_thin_files` to `false`.
- Existing CSV files in `/wardriver/` will be subject to the new threshold the next time auto-upload runs. To upload them regardless, set `min_sweeps_threshold` to `0` or move the files out of `/wardriver/uploaded/thin/` if already quarantined.
