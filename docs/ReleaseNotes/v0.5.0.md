# M5 Cardputer Wardriver — v0.5.0 Release Notes

**Release date:** April 12, 2026
**Target hardware:** M5Stack Cardputer v1.1 (ESP32-S3)
**Firmware binary:** `output/wardriver-0.5.0.bin`

This is the **first public release** of M5 Cardputer Wardriver.

---

## Highlights

- WiGLE CSV v1.6 output — upload directly to wigle.net
- Five live dashboard views with incremental screen updates (no flicker)
- Full web configuration portal accessible from any phone or laptop browser
- Three GPS logging modes — fix-only (strict), zero-GPS (log without coordinates), last-known (reuse stale position)
- Active and passive WiFi scanning with per-channel or all-channel sweep options
- MAC address randomisation with optional OUI spoofing
- WiFi country code and TX power control
- Geofence filtering, SSID/BSSID exclusion lists
- Configurable sound feedback — per-event buzzer options
- Low-battery auto-shutdown with countdown warning
- Graceful shutdown with session summary

## Features

### WiFi Scanning

| Feature | Details |
|---------|---------|
| Scan modes | `active_all`, `active_hop`, `passive_all`, `passive_hop` |
| Active dwell | Configurable per-channel dwell time (default: 150 ms) |
| Passive dwell | Configurable per-channel dwell time (default: 250 ms) |
| Scan delay | Configurable pause between scan cycles (default: 150 ms) |
| MAC randomisation | Rotate device MAC every N scans (`0` = disabled, `1` = every scan) |
| OUI spoofing | Optional — randomised MACs can mimic real manufacturer prefixes |
| WiFi country code | Set ISO 3166-1 code for channel and power compliance (53 countries + Default) |
| WiFi TX power | Configurable in quarter-dBm units: 8–84 (2–21 dBm) |
| Capacity | Tracks up to 10,240 unique BSSIDs per session |

### GPS & Logging

| Feature | Details |
|---------|---------|
| Output format | WiGLE CSV v1.6 |
| GPS logging modes | Three modes for GPS fix loss: `fix_only` (pause logging — cleanest data), `zero_gps` (log with 0,0 coords — for indoor surveys), `last_known` (reuse last position — for brief outages) |
| Accuracy gating | Logging suppressed when HDOP-derived accuracy exceeds threshold |
| Accuracy default | 500 m (configurable) |
| Clock sync | System clock set from GPS — timestamps are accurate without NTP |
| GMT offset | Configurable timezone offset for local time display |
| CSV files | Auto-incrementing filenames: `wardriving_001.csv`, `wardriving_002.csv`, … |
| Write buffering | Buffered writes, flushed every 60 seconds and on shutdown |
| Pause/resume | Press `P` to pause CSV logging mid-session without stopping scanning |

### Display — Five Dashboard Views

Cycle through views with **ENTER**. All views update incrementally — only changed values are redrawn.

| View | Label | Content |
|------|-------|---------|
| A | Summary | Unique APs, total logged, uptime, GPS coordinates, accuracy, altitude, fix type |
| B | Security Snapshot | WPA2 / WPA3 / Open / Other counts, top 3 strongest APs |
| C | Live Feed | Last 10 access points seen across all scans |
| D | New AP Feed | Last 10 brand-new BSSIDs discovered this session |
| E | System Info | Battery level, heap / PSRAM, sound mute status |

### Filtering

- **SSID exclusion list** — comma-separated exact-match SSIDs to suppress from logging
- **BSSID exclusion list** — comma-separated MAC addresses to suppress
- **Geofencing** — up to 10 bounding boxes (top_lat, left_lon, bottom_lat, right_lon); logging suppressed inside zones

### Web Configuration Portal

- Starts automatically on first boot (no config file present) as a WiFi AP (`M5-Wardriver`)
- Accessible during a session by pressing **C** or holding the G0 button at boot
- Single dark-themed page covering all settings: GPS, Scanning, Filters, Hardware, Debug, Web Auth
- HTTP Basic Authentication — configurable username and password
- Auto-closes after 5 minutes of inactivity
- Captive portal detection — Android, iOS, and Windows redirect automatically
- API endpoints: `GET /status` (JSON), `GET /debuglog`, `POST /cleardebuglog`

### Sound Feedback

All sound options are independent and individually configurable:

| Option | Default | Trigger |
|--------|---------|---------|
| Buzzer enabled (master) | On | Overrides all options below when off |
| Keyboard sounds | On | Key presses: view cycle, start/stop, help, config |
| Beep on new SSID | Off | New unique BSSID discovered |
| Beep on blocked SSID | Off | Filtered/excluded SSID scanned |
| Sound in geofenced area | Off | Periodic 10-second beep while inside a geofence |
| Buzzer frequency | 2500 Hz | Configurable: 100–8000 Hz |
| Buzzer duration | 50 ms | Configurable: 10–500 ms |

Press **S** to mute/unmute all sounds at any time during a session.

### Status LED

SK6812 RGB LED (GPIO 21) shows device state at a glance — see [LED Indicator](../led-indicator.md).

| Colour | State |
|--------|-------|
| Yellow | Searching for GPS satellites |
| Blue | Active scanning |
| Green | GPS fix acquired, ready |
| Purple | Config Mode |
| Red | Error or low battery |

### Power & Battery

- Battery level shown in the footer of every dashboard view (`B=XX%`)
- **Low battery auto-shutdown** — countdown warning at configurable threshold (default: 5%)
- Display blanking via **B** key — reduces power draw without halting scanning
- Scan stop/start via **X** key — halts WiFi scanning while keeping GPS active

### Keyboard Controls

| Key | Action |
|-----|--------|
| `ENTER` | Cycle dashboard views (A → B → C → D → E) |
| `H` | Show/hide keyboard help overlay |
| `B` | Toggle display blank |
| `C` | Enter Config Mode |
| `S` | Toggle sound mute |
| `Q` | Safe shutdown with session summary |
| `X` | Stop/start WiFi scanning |
| `P` | Pause/resume CSV logging |
| `G0 button` | Boot: force Config Mode · Running: drop FLAG marker · Shutdown: reboot |

### Shutdown & Session Summary

Pressing **Q** (or triggering low-battery shutdown) flushes all buffers, unmounts the SD card, and shows a session summary:

- Total rows logged
- Total unique APs seen
- Session duration
- Total scan sweeps
- Peak APs per sweep
- Average scan time (ms)

### Debug & Diagnostics

| Feature | Details |
|---------|---------|
| Debug logging | Optional CSV log to `/wardriver/logs/debug.log` |
| Super debug | Verbose per-scan instrumentation (channel results, MAC changes, timing) |
| System stats | Periodic heap, PSRAM, battery, and station-count logging (default: every 5 min) |
| Debug log viewer | Browse the debug log from your browser at `/debuglog` (when debug is enabled) |
| Serial output | Always active at 115200 baud regardless of SD debug setting |

## Configuration File

Config is stored at `/wardriver/wardriverconfig.txt` on the SD card as JSON.

A full example with all defaults is provided in [`config.txt.example`](../../config.txt.example) at the project root.

The device enters Config Mode automatically if the config file is missing or unparseable.

---

## SD Card Layout

```
/wardriver/
  wardriverconfig.txt     ← JSON config file
  wardriving_001.csv      ← First session WiGLE CSV
  wardriving_002.csv      ← Second session (auto-increment)
  logs/
    debug.log             ← Debug log (when debug.enabled = true)
```

## Known Limitations

- WiFi scanning is 2.4 GHz only (5 GHz not supported by ESP32-S3 hardware)
- Passive scan modes yield fewer results than active modes on networks with infrequent beacons
- GPS cold-start fix time is 30–90 seconds with a clear sky view; logging will not begin until a fix is acquired in `fix_only` mode
- The web portal occupies WiFi — scanning is paused while Config Mode is active
